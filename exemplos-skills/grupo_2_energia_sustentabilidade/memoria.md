# Memória Simulada: Otimização de Consumo Energético

## Preferências do usuário
- O usuário prefere recomendações práticas para gestor predial.
- O usuário quer sempre considerar conforto antes de economia.
- O usuário pediu estimativas simples: impacto baixo, médio ou alto.
- O usuário prefere que a resposta diferencie simulação de resultado real.

## Decisões já aprendidas
- Salas ocupadas não devem receber recomendação de desligamento total de ar-condicionado sem análise.
- Ajuste de temperatura deve ser sugerido de forma gradual.
- Horários de pico devem aparecer na justificativa.
- Quando houver dados incompletos, informar hipóteses antes da recomendação.

## Histórico de interações simuladas
### Interação 1
Usuário perguntou sobre sala de aula com ar-condicionado ligado fora do horário.
Resposta útil foi recomendar automação por agenda e validação com ocupação real.

### Interação 2
Usuário perguntou sobre laboratório com consumo alto e baixa ocupação.
Resposta útil foi sugerir setpoint mais alto, desligamento parcial e medição por uma semana.

### Interação 3
Usuário pediu relatório para diretoria.
Resposta útil foi apresentar diagnóstico, ação, impacto esperado e risco para conforto em linguagem gerencial.

## Padrão de resposta preferido
Usar sempre:

1. diagnóstico;
2. oportunidade de economia;
3. recomendação;
4. impacto esperado;
5. conforto;
6. dados faltantes.

## Termos importantes
- consumo energético;
- horário de pico;
- conforto térmico;
- ocupação;
- ar-condicionado;
- iluminação;
- tarifa energética;
- simulação;
- economia estimada.

## Restrições
- Não prometer economia exata sem base de cálculo.
- Não priorizar economia acima de conforto mínimo.
- Não sugerir intervenção elétrica sem profissional qualificado.
- Não tratar dados simulados como medição real.
